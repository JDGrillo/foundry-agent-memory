## Description
Build a cleanup utility that removes memory stores and agents created by the demo scenarios. Prevents orphaned resources in Azure subscriptions. Supports selective cleanup by scenario and a confirmation prompt for safety.

## Acceptance Criteria
- [ ] AC-1: `cleanup.py` at project root removes all demo memory stores and agents
- [ ] AC-2: Running without flags lists demo resources and prompts for confirmation
- [ ] AC-3: `--scenario <name>` deletes only that scenario's stores and agent (helpdesk, standup, cs)
- [ ] AC-4: `--all` deletes all demo resources without prompting (for CI/automation)
- [ ] AC-5: No error if resources don't exist (idempotent)
- [ ] AC-6: Only deletes resources in the known demo list (never touches other stores/agents)
- [ ] AC-7: Confirmation prompt defaults to "No"

## Technical Notes
- Store-to-scenario mapping:
  ```python
  SCENARIO_STORES = {
      "helpdesk": ["helpdesk-memory"],
      "standup": ["standup-memory"],
      "cs": ["cs-user-memory", "cs-team-memory"],
  }
  ```
- Also delete agents by name (list agents, filter by known demo agent names)
- Use `argparse` for CLI interface
- Use `store_manager.delete_store()` from DEV-003 for memory store deletion
- Use `agents_client.delete_agent()` for agent cleanup
- Print confirmation for each deleted resource
