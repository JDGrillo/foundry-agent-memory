## Status: Backlog

## Priority: P1

## Description
Implement a utility script that queries and displays memory store contents via the SDK. After running a scenario, demo engineers use this to prove memory persistence by showing stored profiles, chat summaries, and scoped entries — either in addition to or as a fallback for portal inspection.

## Acceptance Criteria
- [ ] AC-1: `inspect_memory.py` at project root queries and displays memory store contents
- [ ] AC-2: Shows all scopes within a store (e.g., `user_123`, `team-alpha`, `account-acme`)
- [ ] AC-3: Shows user profile data for stores with `user_profile_enabled`
- [ ] AC-4: Shows chat summaries for stores with `chat_summary_enabled`
- [ ] AC-5: Supports `--store <name>` flag for selective inspection (default: all demo stores)
- [ ] AC-6: Supports `--scope <id>` flag to inspect a specific scope
- [ ] AC-7: Output formatted for demo readability (clear headers, structured data)

## Technical Notes
- Use `AIProjectClient` memory store APIs to query stored data
- API calls might include:
  - List memory entries by scope
  - Get user profile for a scope
  - Get chat summaries for a scope
- Format output with clear sections per store → per scope → entries
- Truncate long content with option to show full (`--verbose`)
- If SDK doesn't expose direct memory entry listing, document what IS available and show that
- Use console formatter (DEV-006) for output styling
