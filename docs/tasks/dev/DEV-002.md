## Description
Implement the environment configuration module that loads `.env` variables, validates all required values are present, and provides a typed configuration object for use by other modules. Must produce clear, actionable error messages when configuration is missing or invalid.

## Acceptance Criteria
- [ ] AC-1: `src/common/env.py` loads variables from `.env` using `python-dotenv`
- [ ] AC-2: Validates that `PROJECT_CONNECTION_STRING`, `CHAT_MODEL_DEPLOYMENT`, and `EMBEDDING_MODEL_DEPLOYMENT` are set
- [ ] AC-3: Missing variables produce error messages naming the specific variable and referencing `.env.example`
- [ ] AC-4: Auth failures suggest running `az login` or checking credential setup
- [ ] AC-5: Exposes a config object or function that other modules import (e.g., `get_config()`)
- [ ] AC-6: Config includes the project connection string parsed into usable components if needed

## Technical Notes
- Use `DefaultAzureCredential` from `azure-identity` for authentication
- The `AIProjectClient` is constructed from the connection string — consider whether to expose the client instance here or just the config values
- Keep it simple: a dataclass or named tuple for config, a `load_config()` function that validates and returns it
- Fail fast at startup — don't let scenarios run halfway before discovering missing config
