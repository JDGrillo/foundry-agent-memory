## Description
Implement the Customer Success scenario demonstrating hybrid per-user + group memory. This is the most complex scenario: it uses TWO memory stores simultaneously — one for individual CSM context (per-user) and one for shared account intelligence (group). Demonstrates seamless handoff between team members with zero context loss.

The agent is created via Foundry Agents Service with two `MemorySearchTool` instances attached. All conversations run via threads/runs API and are portal-visible.

## Technical Notes
- Memory store scoping:
  - `cs-user-memory`: scope = `csm_sarah` or `csm_mike` (per-CSM)
  - `cs-team-memory`: scope = `account-acme` (shared across all CSMs for that account)
- The agent has dual MemorySearchTool — it searches both stores on each interaction
- Simulation flow:
  1. Sarah logs interaction with Acme Corp (expansion to 500 seats, SSO requirement by Q3)
  2. Sarah adds more account intel (CTO stakeholder, onboarding concern)
  3. Mike takes over: "I'm covering for Sarah on Acme Corp. What's the latest?"
  4. Agent retrieves account context from cs-team-memory, personal context from cs-user-memory
  5. Mike asks: "What's our full history with Acme Corp?" — agent synthesizes
- Use console formatter (DEV-006) to distinguish which memory store is being read/written
