## Description
Implement the shared memory store manager module that provides CRUD operations for Foundry Memory Stores. This is the central utility used by all scenarios for store provisioning and teardown. Stores created via this module are portal-visible (REQ-F-035) by nature of using the `client.beta.memory_stores.create()` API.

## Acceptance Criteria
- [ ] AC-1: `src/memory/store_manager.py` implements `create_store(name, config)` — creates or returns existing store (idempotent)
- [ ] AC-2: Implements `delete_store(name)` — removes store and all its data
- [ ] AC-3: Implements `delete_scope(store_name, scope_id)` — removes a specific scope's data
- [ ] AC-4: Implements `list_stores()` — lists all memory stores in the project
- [ ] AC-5: Running `create_store()` twice with the same name produces no error or duplicate
- [ ] AC-6: `src/memory/config.py` defines store configuration presets for each scenario (embedding model, chat_summary_enabled, user_profile_enabled, user_profile_details)
- [ ] AC-7: Console output confirms store creation with store name and ID
- [ ] AC-8: Created stores are visible in the Azure AI Foundry portal

## Technical Notes
- Use `AIProjectClient` from `azure-ai-projects` for memory store operations
- API: `client.beta.memory_stores.create()`, `client.beta.memory_stores.list()`, `client.beta.memory_stores.delete()`
- Store config presets per architecture doc:
  - `helpdesk-memory`: chat_summary=true, user_profile=true, profile_details="role, department, OS, device model, RAM, software versions, past issues and resolutions, communication preference"
  - `standup-memory`: chat_summary=true, user_profile=false
  - `cs-user-memory`: chat_summary=true, user_profile=true, profile_details="interaction style, client portfolio, specialization areas"
  - `cs-team-memory`: chat_summary=true, user_profile=false
- Idempotency: list stores first, check by name, create only if not found
- Embedding model deployment name comes from config (DEV-002)
