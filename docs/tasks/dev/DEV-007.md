## Description
Implement the IT Help Desk scenario script demonstrating per-user memory. The script provisions the `helpdesk-memory` store, creates a portal-visible agent via the Foundry Agents Service, and runs two sessions showing memory in action:
1. First contact — no prior memory, agent gathers user context, resolves issue
2. Return visit — agent recalls user profile and prior issue, provides contextual response

The agent and all conversation threads must be visible in the Azure AI Foundry portal.
