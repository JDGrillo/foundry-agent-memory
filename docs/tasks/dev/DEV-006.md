## Description
Implement a console output formatting utility that visually distinguishes user messages, agent responses, and memory events in terminal output. This is critical for demo readability — viewers must be able to SEE memory working.

## Acceptance Criteria
- [ ] AC-1: Visual distinction between user messages, agent responses, and memory events (color or formatting)
- [ ] AC-2: Memory retrieval events labeled clearly (e.g., "MEMORY RETRIEVED:")
- [ ] AC-3: Retrieved memory content displayed (truncated if >200 chars)
- [ ] AC-4: "No memory found" explicitly shown on first interactions
- [ ] AC-5: Memory write events confirmed (e.g., "MEMORY STORED")
- [ ] AC-6: Output readable at 120-char terminal width
- [ ] AC-7: Clear visual separation between conversation turns

## Technical Notes
- Use ANSI color codes or the `rich` library (if added to deps) for formatting
- Prefer simple ANSI codes to avoid adding dependencies; or add `rich` if it significantly improves output
- Provide functions like:
  - `print_user_message(user_id, message)`
  - `print_agent_response(message)`
  - `print_memory_event(event_type, content)`
  - `print_section_header(title)`
- Keep it minimal — this is a formatting layer, not logging
- Must work on Windows terminal (no Unix-only escape codes)
