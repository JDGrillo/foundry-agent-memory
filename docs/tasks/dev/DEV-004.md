## Description
Implement the agent definitions module using the **Foundry Agents Service API** (`azure-ai-agents` SDK). Agents are created via `project_client.agents.create_agent()` so they are registered server-side and visible in the Azure AI Foundry portal. Each agent has `MemorySearchTool` attached referencing the appropriate memory store(s). This module also provides helper functions for creating threads, sending messages, and running agent responses via the threads/runs API.

This is the core architectural change from the previous plan: agents are NOT created locally via `agent_framework.Agent` — they are created as Foundry-hosted agents with portal visibility.

## Technical Notes
- SDK pattern:
  ```python
  from azure.ai.agents import AgentsClient
  from azure.ai.agents.models import MemorySearchTool

  agents_client = AgentsClient(endpoint=..., credential=...)
  
  tool = MemorySearchTool(memory_store_id=store.id)
  agent = agents_client.create_agent(
      model=chat_model_deployment,
      name="IT Help Desk Agent",
      instructions="...",
      tools=[tool]
  )
  ```
- Thread/run pattern:
  ```python
  thread = agents_client.threads.create()
  agents_client.messages.create(thread_id=thread.id, role="user", content="...")
  run = agents_client.runs.create_and_process(thread_id=thread.id, agent_id=agent.id)
  messages = agents_client.messages.list(thread_id=thread.id)
  ```
- For Customer Success, attach two MemorySearchTool instances with different store IDs
- Consider providing a `get_or_create_agent()` pattern for idempotency
- Thread creation should accept a scope/user_id to enable proper memory scoping
